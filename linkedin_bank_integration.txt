Quick post about a side project I shipped end-to-end.

Meet Bundy (bundy.ro) — named after Al Bundy from Married with Children, the world's most famously broke shoe salesman. An expense tracker felt fitting.

Mobile-first PWA installable on iOS/Android from the browser (no App Store). 9 currencies with BNR FX conversion at each transaction's own date — a "50 EUR Netflix from January" stays equivalent to January's rate forever, no monthly drift. Automated subscriptions + loans. Personal vs business-card split. Dark mode, offline, EN+RO. $0/month hosting.

I wore every hat: PM scoping, designer making UX trade-offs, fullstack from DB schema to iOS PWA install flow.

Highlights:
• Fast entry: Quick (one-tap stepper), Predefined (form pre-fill), Fixed (FAB).
• Recurring engine runs client-side at boot AND via Vercel Cron at 00:15 UTC, sharing the same idempotent generator.
• Analytics: 6-month trend, weekly bars locked within calendar months, category donut, drill-down filters.
• Hidden expenses behind a 4-digit PIN with sliding TTL — stops shoulder-surfing, the actual threat model.
• Company-card toggle: splits totals personal vs business across Home / Expenses / Analytics.
• Daily DB backup via GitHub Actions through Supabase's IPv4 pooler (direct connection is IPv6-only, GH runners don't speak IPv6).
• Bilingual EN/RO via react-i18next, browser-detected, persisted to profile.

I also spent days on EU Open Banking. GoCardless rejected new signups; Salt Edge sandbox worked but production needs AISP licensing + a legal entity. Dropped it. Lesson: check regulation before architecting.

Stack: Vite + React 18 + TS + Mantine v7, TanStack Query, RHF + Zod, Supabase (Postgres + Auth + RLS), Vercel, Resend. Vitest, 117 tests.

Live at bundy.ro.
